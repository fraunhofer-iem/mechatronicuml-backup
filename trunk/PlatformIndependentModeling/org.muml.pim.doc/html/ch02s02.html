Getting Started

This chapter gives an instruction to install the MechatronicUML Tool Suite within Eclipse.

Preconditions

MechatronicUML Tool Suite needs Java 1.6 for full feature support. If Java 1.6 is not yet on your computer: please visit http://www.oracle.com/technetwork/java/ ​ and download the latest Java JDK 1.6 release (SE aka Standard Edition) for your platform (install JDK and not the JRE, if possible).

Important

Make sure the Java 1.6 is the default Java platform implementation on your platform (for example, on Windows, open a command-line window and type "java -version" into it, without the double quotes).

Installing Eclipse And Basic Modeling Plugins

  1. Download and install the Eclipse Modeling Tools (Indigo-Release): http://www.eclipse.org/downloads/packages/eclipse-modeling-tools/indigor

  2. Be sure that you choose Java 6 as default Java Runtime. Window -> Preference -> Java -> Installed JREs -> JRE6 must be checked. The common windows path (32Bit) is "C:\Program Files\Java\jre6".

    Figure 2.1. JRE 6 must be checked.


  3. Install the Subversive-Plugin to be able to checkout and commit to repositories with SVN:

    1. Click Help -> Install New Software...

      Figure 2.2. Install New Software...


    2. Select Site "Indigo - ​http://download.eclipse.org/releases/indigo"

      Figure 2.3. Select Indigo.


    3. Choose the following plugin: *Category "Collaboration": Subversive SVN Team Provider (Incubation)

      Figure 2.4. Choose the Subversive SVN Team Provider.


    4. Press Next and follow the installation instructions.

  4. After the restart of Eclipse, Subversive will prompt you to install an SVN Adapter. Please choose a propriate adapter for your plattform and install the required plugins.

    Figure 2.5. Choose a SVN adapter.


  5. Install required modeling components with the "Installing modeling components" wizard: ModelingComponents

    1. click on the button "Installing modeling components"

    2. select from the component list:

      • Graphical Modeling Framework Tooling

      • XPand

      • XText

      • Operational QVT

      • Extended Editing Framework

      Figure 2.6. Choose these components.


    3. Click "Finish" and follow the instructions.

  6. Install Eclipse Edapt (all features provided in the Update-Site at http://download.eclipse.org/edapt/update-site ​). Edapt enables us to migrate old fujaba-files each time our Meta-Model changes. If you want to apply changes to the meta-model as well, you have to use the Edapt operations so that Edapt can track your changes.

  7. OPTIONAL: For layouting GMF-based diagrams, you have to install KIELER. Its short for Kiel Integrated Environment for Layout and is an Eclipse-Plugin that has better layouting algorithmes than the ones shipped with Eclipse. We mainly use Kieler for layouting our EMF Meta Model diagrams, but you can also use Kieler to layout any other GMF-based diagrams like the MUML Real-Time Statechart Diagram.

    • Click Help -> Install New Software... -> Click on Add...

    • Choose a name and copy the following link to Location and click OK: ​ http://rtsys.informatik.uni-kiel.de/~kieler/updatesite/release_0.6/

    • Install the wanted plug-ins.

  8. Check for updates by clicking on Help -> Check for Updates.

Installing Plugins of the MechatronicUML Tool Suite

Our Main-MechatronicUML-SVN is at: ​https://svn-serv.cs.upb.de/mechatronicuml (anonymous login for checkout; IMT-Account and write access needed for checkin). Check-Out:

  • Switch to folder "trunk"

  • Mark all Subfolders

  • Right mouse click --> choose "Check Out"

    Figure 2.7. That's how you check out.


Additionally you need some plugins from another SVN (org.muml.modeling): http://svn.codespot.com/a/eclipselabs.org/sdm-commons (anonymous login for checkout; Google-Code-Account and write access needed for checkin) Check-Out:

  • Switch to folder "trunk/plugins"

  • Mark all Subfolders

  • Right mouse click --> choose "Check Out"

Since we do not need all the plug-ins from the storydriven-SVN, you may close all unrelated projects as described below:

  • Within the Package Explorer mark all the projects from the mechatronicuml-SVN.

  • Right click on on of the projects and select "close unrelated projects" and confirm with OK.

    Figure 2.8. That's how you close unrelated projects.


  • In the following screenshot all the selected projects are the current required projects from the storydriven-SVN.

    Figure 2.9. That's how you close unrelated projects.


Run Configurations

Be sure to add the following vmarks to your run configuration! -XX:PermSize=256M -XX:MaxPermSize=512M

  • Switch to java Perspective

  • Right click on one of the projects

  • select Run As -> Run Configuration...

    Figure 2.10. Select Run As -> Run Configuration...


  • Add the following vmarks to your run configuration! -XX:PermSize=256M -XX:MaxPermSize=512M

    Figure 2.11. Add these arguments.


Otherwise you probably get the following error at your eclipse console. !ENTRY org.eclipse.ui 4 0 2011-06-28 14:57:41.478 !MESSAGE Unhandled event loop exception Error while logging event loop exception: java.lang.OutOfMemoryError?: PermGen? space See also: http://wiki.eclipse.org/FAQ_How_do_I_increase_the_permgen_size_available_to_Eclipse%3F ​

Starting FujabaRT

Start FujabaRT after checking out the sources:

  • In Eclipse, switch to Java perspective.

  • Select one of the projects.

  • Press the "Run"-button (green circle with white triangle)

  • It might happen, that some projects do not compile. Do not pay attention to that, nevertheless it should work.
